Grids

Create a fully responsive, fluid and nestable grid layout.

The grid system of NGkit follows the mobile-first approach and accomodates up to 10 grid columns. It uses units with predefined classes inside each grid, which define the column width. It is also possible to combine the grid with classes from the Flex component, although it works only in modern browsers.

Usage

To create the grid container, add the .ng-grid class to a parent element. Add one of the .ng-width-* classes to child elements to determine how the units will be sized. The grid supports 1, 2, 3, 4, 5, 6 and 10 unit divisions. This table gives you an overview of the .ng-width-* classes that can be applied to units.

Class Description
.ng-width-1-1 Fills 100% of the available width.
.ng-width-1-2 Divides the grid into halves.
.ng-width-1-3 to .ng-width-2-3Divides the grid into thirds.
.ng-width-1-4 to .ng-width-3-4Divides the grid into fourths.
.ng-width-1-5 to .ng-width-4-5Divides the grid into fifths.
.ng-width-1-6 to .ng-width-5-6Divides the grid into sixths.
.ng-width-1-10 to .ng-width-9-10Divides the grid into tenths.

We built an intentional redundancy into each set of unit classes, so that for instance the .ng-width-5-10 class will work just as well as .ng-width-1-2.

Responsive width

NGkit provides a number of very useful responsive widths classes. Basically they work just like the usual width classes, except they are prefixed, so that they only come to effect at certain breakpoints. These classes can be combined with the visibility classes from the Utility component. This is great to adjust your layout and content for different device sizes.

ClassDescription
.ng-width-* Affects all device widths, grid columns stay side by side.
.ng-width-small-* Affects device widths of 480px and higher. Grid columns will stack on smaller sizes.
.ng-width-medium-* Affects device widths of 768px and higher. Grid columns will stack on smaller sizes.
.ng-width-large-* Affects device widths of 960px and higher. Grid columns will stack on smaller sizes.

IMPORTANT To create a margin between stacking grid columns, just add the data-ng-grid-margin attribute.

Source ordering

You can change the display order of the columns to keep a specific column order in the source code. Add one of the .ng-push- classes to move the column to the right and add one of the .ng-pull- classes to move a column to the left. This allows you for example to flip the columns' display order for wider viewports. The classes can also be used to offset columns, creating additional space between them.

Source ordering is useful for SEO and responsive design, because in narrow viewports the grid will be displayed according to the source order of the markup.

NOTE This feature only works in combination with one of the .ng-width-medium-* classes.

Example
.ng-width-medium-1-2 .ng-push-1-2
.ng-width-medium-1-2 .ng-pull-1-2
.ng-width-medium-2-5 .ng-push-3-5
.ng-width-medium-2-5 .ng-pull-2-5
