Filtering

Using a data source, you can create inputs that allow users to filter a list of pre-generated values while typing.

Basic Usage

This component requires three things: a text input, a wrapper to contain everything, and a data source. Optionally you can add your own container for the results. If you do not, a <div class="ng-panel ng-scrollable-box ng-filter-target"></div> is inserted into the wrapper.

Add the .ng-filter class to the wrapper (typically a <div> element). To enable the necessary JavaScript for the filter input, you also need to add the data-ng-filter attribute. Add {source:'PATH/TO/RESULTS'} to the data attribute.

A minimalist markup looks like:

<div class="ng-filter" data-ng-filter="{source: '/api/query/me'}">
    <input type="text">
    <div class="ng-filter-target">
        <!-- this is optional, but included to show its minimum markup -->
    </div>
</div>

Data Sources

Filtering accepts a function, object or string as its source configuration.

Your filtering function should accept one argument: a release function. The release function accepts an array of objects and renders them. Your filtering function should create an array based on the input text and call the release function.

For object datasources, you must have a value attribute. The filtering component case-insensitively tests the input against that attribute. The default template also uses html_url and title attributes. You can customize the template (see below).

When a string is passed as its source, the filter component treats it as a URL and makes an AJAX call to it. You can also pass in other configuration options that modify this behavior:

ParameterDefaultDescription
minLength1The minimum length of characters that must exist before searching the data source. An empty input will always query for all items.
param'search'The parameter to pass to an AJAX search query
method'get'The method (GET or POST) to use for the AJAX search query
delay300How long, in milliseconds, to wait for another keyup event before performing the search
loadingClass'ng-loading'The class to apply to the results container while loaing
renderernullA custom renderer function to handle the results. By default it uses the default template and renders it into the results container

All the examples below use an array, defined in the first example, for their data source.

Custom template

The filter and autocomplete components use a simplified template language.

###Template Markup:

MarkupMeaning
{{~variable}} ... {{/variable}} Iterate over every object in an array. Within the block, $item is the value of the current array item.
{{:variable}} ... {{/variable}} Iterate over every property in an object. Within the block, $key is the name of the property, and $val is its value
{{#variable}} ... {{/variable}} If variable evaluates as true (see what evaluates as true) then execute the block.
{{^variable}} ... {{/variable}} If variable evaluates as false (see what evaluates as true) then execute the block.
{{!variable}} Insert the value of variable without escaping.
{{variable}} Insert the escaped value of variable.

###Defining the template to use

The filter component looks for a <script> tag of type text/filter within the .ng-filter container. If found, it is automatically used.

You can also pass in a template parameter in the data-ng-filter attribute.

Filter Pickers

In some cases, you might want to provide an alphabetical selector as well as an text input. Simply give a <div> tag the class .ng-filter-picker and wrap the choices in <a> tags. Each <a> tag should have a data-value attribute with the value for that choice.

When a user clicks on a choice, the data-value is entered into the text input and a search is triggered as if they typed it in themselves.
